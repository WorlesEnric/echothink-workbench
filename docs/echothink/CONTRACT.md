# Echothink-Workbench — Canonical Engineering Contract

> This document is the **single source of truth** that every package and every Codex
> worker MUST conform to. It defines the shared vocabulary, package boundaries, file
> formats, and TypeScript interfaces so that independently-built packages compose
> cleanly. It is derived from `workbench_spec.md` (the architecture spec). Where the
> spec gives an example shape, this document fixes the exact, typed contract.
>
> **Founding rule:** *Schema controls the domain. Code controls the experience.
> Runtime controls the boundary.*

---

## 0. Naming, scope, and toolchain

- **Workbench-owned packages** use the npm scope **`@echothink/*`** (e.g.
  `@echothink/domain-manifest`). These are the packages this project builds.
- **The pre-existing UI component library** is **`@echothink-ui/*`** (Carbon-based,
  e.g. `@echothink-ui/core`, `@echothink-ui/data`, `@echothink-ui/task`). The workbench
  **never modifies** it; it describes it via a sidecar registry and imports from it.
- **App Domains** generated by the workbench import only `@echothink/app-domain-sdk`
  and approved `@echothink-ui/*` packages. They never import runtime internals.

### Toolchain (fixed for all `@echothink/*` packages)
- TypeScript **strict** mode, `"module": "NodeNext"`, `"target": "ES2022"`, ESM (`"type": "module"`).
- Validation/parse: **zod** (v3). YAML: **yaml** (`yaml` npm package). Hashing: Node `crypto` (sha256).
- Tests: **vitest**. Every package has `src/`, `src/index.ts` (public surface), `*.test.ts` colocated, `tsconfig.json`, `package.json`, `README.md`.
- No default exports. Named exports only. No `any` in public signatures; use `unknown` + zod.
- Each package builds with `tsc` (declaration + sourceMap) to `dist/`. Dev consumption is via tsconfig path aliases (see §15).
- Determinism: no `Date.now()`/`Math.random()` inside compile/hash/codegen paths — callers inject clocks/ids. Canonical JSON serialization (sorted keys) for any hashed artifact.

### Monorepo layout (inside `echothink-workbench/`)
```
packages/@echothink/
  domain-manifest/      # schema, parser, compiler, codegen, validators
  app-domain-sdk/       # frontend + backend SDK, effect contracts, preview stubs
  app-domain-runtime/   # runtime authority + preview runtime
  ui-registry/          # @echothink-ui/* metadata, recipes, surface templates
  validation/           # gate runners + validate:domain CLI + report model
  agent-harness/        # Codex runner, policies, patch collection, repair loop
  surface-factory/      # manifest -> standard surface generator
  registry/             # App-Domain registry + promotion + release manifest
  shared-types/         # cross-package primitive types (no logic, no deps)
domains/                # generated/example App Domain artifacts (e.g. github-triage)
docs/echothink/         # this contract + design notes
```
`@echothink/shared-types` holds ONLY the primitive shared types in §1 so packages do
not create import cycles. Everything else imports from it.

## 2. The App-Domain Manifest (`@echothink/domain-manifest`)

The manifest is YAML authored by humans/agents; it is the human-readable source of
truth. The Zod schema below is the authoritative shape. It is a faithful superset of
`workbench_spec.md` §9 and §20. The `github-triage` example in the spec MUST validate.

### 2.1 Top-level structure (Zod schema names in parentheses)
```yaml
apiVersion: echothink.ai/v1          # const
kind: AppDomain                      # const
metadata: { id, name, owner, version, sdkContractVersion, uiRegistryVersion, description? }
identity: { roles: Role[], personas?: Persona[] }
entities: { <EntityName>: Entity }   # record keyed by PascalCase entity name
queries: { <queryId>: Query }        # queryId is dotted, e.g. issue.openQueue
permissions: Permission[]
unitProcesses: { <processId>: UnitProcess }
events: { <eventId>: EventDef }
effects: { <effectId>: EffectDef }
surfaces: Surface[]
release: ReleaseSpec
```

### 2.2 Field contracts
- **metadata**: `id` kebab-case `^[a-z][a-z0-9-]*$`; `version` semver; `sdkContractVersion` & `uiRegistryVersion` strings.
- **Role**: `{ id: string; name?: string; description?: string; assignable?: boolean; delegatable?: boolean }`. `id` snake/kebab lower.
- **Persona** (preview fixtures): `{ id; role: roleId; tenantId?; label?; invalid?: boolean }`.
- **Entity**:
  ```ts
  {
    key: string;                       // storage key, lower
    schema: Record<string, FieldSpec>; // field name -> spec (see below)
    tenantScope: "workspace" | "organization" | "system";
    stateField?: string;               // which field is the lifecycle state (enum)
    stateMachine?: { initial: string; transitions: Array<{from: string; to: string; via?: processId}> };
    relationships?: Record<string, { entity: string; cardinality: "one"|"many" }>;
    sensitivity?: string[];
    retention?: string;
    audit?: { read?: AuditLevel; write?: AuditLevel };
  }
  ```
- **FieldSpec**: either a shorthand string (`"string" | "number" | "boolean" | "date" | "json"`),
  a nullable shorthand (`"string?"`), or an object:
  ```ts
  | string
  | { type: "string"|"number"|"boolean"|"date"|"json"; optional?: boolean }
  | { enum: string[]; optional?: boolean }
  | { type: "string[]"|"number[]"; optional?: boolean }   // arrays
  | { ref: string; optional?: boolean }                    // reference to entity key
  ```
  The compiler normalizes all forms into a canonical `NormalizedField`
  `{ name; kind: FieldKind; optional: boolean; enumValues?: string[]; arrayOf?: FieldKind; refEntity?: string }`.
- **Query**:
  ```ts
  { entity: string; filter?: Record<string, unknown>; sortableBy?: string[];
    permissions?: { read?: permissionId }; pageable?: boolean }
  ```
- **Permission**: `{ id: string; roles: roleId[]; description?: string }`. `id` dotted, e.g. `issue.triage`.
- **UnitProcess**:
  ```ts
  {
    input: Record<string, IoFieldSpec>;     // same FieldSpec grammar incl. enum(...) shorthand
    output?: Record<string, IoFieldSpec>;
    requires?: { permission?: permissionId };
    preconditions?: string[];                // human-readable + machine predicates "Entity.field op value"
    reads?: string[];                        // entity names
    writes?: string[];                       // entity names
    transitions?: Record<string, // "Entity.field"
      { from?: string; to?: string } | Record<string /*inputValue*/, string /*toState*/]>;
    emits?: eventId[];
    effects?: effectId[];                     // declared effects this process may invoke
    audit?: { level: AuditLevel; reasonRequired?: boolean };
    idempotency?: { key: string };            // expression over input + processRunId
    compensation?: processId;
    actorType?: "human" | "system" | "ai" | "mixed";
    policyClass?: PolicyClass;
  }
  ```
  Note the `transitions` value supports BOTH the `{from,to}` form (spec §9) and the
  input-keyed map form (spec §20 `dataset.reviewSample`). Normalize to a discriminated union.
- **EventDef**: `{ payload: Record<string, IoFieldSpec> }`.
- **EffectDef** (spec §11):
  ```ts
  {
    owner?: string; version?: string;
    input?: Record<string, IoFieldSpec>; output?: Record<string, IoFieldSpec>;
    requiredPermission?: permissionId;
    secretRef?: string;                      // ID only, NEVER a value
    egress?: { allowHosts: string[] };
    audit?: { redact?: string[]; level?: AuditLevel };
    idempotency?: { key: string };
    rateLimit?: { perMinute?: number; perHour?: number };
    retry?: { maxAttempts?: number };
    environments?: Environment[];
    frontendInvocable?: boolean;             // default false (spec §10 default rule)
  }
  ```
- **Surface**:
  ```ts
  {
    id: string; type: SurfaceType; route: string;
    requiredPermissions?: permissionId[];
    // standard:
    page?: string;                            // ui-registry page-template id, e.g. "EntityTable"
    query?: queryId;
    bindings?: Record<string, unknown>;       // page-template-specific data bindings
    // composed/custom:
    entry?: string;                           // path to index.tsx
    allowedImports?: string[];                // import allowlist (composed/custom)
    // custom only:
    exceptionFile?: string;                   // path to custom-surface.exception.yaml
    isolation?: "none" | "iframe" | "worker";
  }
  ```
- **ReleaseSpec**: `{ channel: ReleaseChannel; requiredApprovals: Array<ApprovalRole | "security-if-effects-changed"> }`.

Canonical compiled manifest. Shape = `CompiledManifest` serialized via
`canonicalJSONStringify`, plus top-level `lockfileVersion: 1` and `manifestDigest`.

### 2.5 `capability-map.json`
```ts

```

### 2.6 Generated kernel TypeScript (the SDK contract surface)
`generated-types.ts` exports a single domain descriptor type, e.g.:
```ts
export interface GitHubTriageDomain {
  id: "github-triage";
  entities: { Issue: { id: string; repo: string; title: string;
    state: "open"|"triaged"|"assigned"|"closed"; labels: string[];
    assignee: string | null; priority: "low"|"medium"|"high"|"urgent" } };
  queries: { "issue.openQueue": { args: {/*filter overrides*/}; row: Issue } };
  processes: { "issue.triage": { input: {...}; output: {...} } };
  events: { "issue.triaged": { issueId: string; priority: string; actorId: string } };
  effects: { "github.issue.comment": { input: {...}; output: {...} } };
  permissions: "issue.read" | "issue.triage" | "issue.assign";
}
```
The interface name is `PascalCase(metadata.id) + "Domain"`. This type is the generic
parameter to `useAppDomain<T>()`. Codegen MUST be deterministic and prettier-free
(emit already-formatted code).

## 3. App-Domain SDK (`@echothink/app-domain-sdk`)

The SDK is the ONLY way generated UI code touches domain state/effects. It is typed
against a domain descriptor `D` (the generated `*Domain` interface from §2.6).

### 3.1 Frontend SDK
```ts
export interface AppDomainClient<D extends DomainDescriptor> {
  identity: { current(): IdentityContext };
  permissions: { can(permission: PermKey<D>, ctx?: PermCheckCtx): boolean };
  entities: {
    query<Q extends QueryKey<D>>(q: Q, args?: QueryArgs<D, Q>): Promise<QueryRow<D, Q>[]>;
    get<E extends EntityKey<D>>(entity: E, id: string): Promise<EntityShape<D, E> | null>;
  };
  processes: {
    run<P extends ProcessKey<D>>(p: P, input: ProcessInput<D, P> & { reason?: string }): Promise<ProcessResult<D, P>>;
    canRun<P extends ProcessKey<D>>(p: P): boolean;
  };
  events: { subscribe<E extends EventKey<D>>(e: E, cb: (payload: EventPayload<D, E>) => void): () => void };
  audit: { annotate(input: { target: string; reason: string }): Promise<void> };
  effects: { invoke<F extends EffectKey<D>>(f: F, input: EffectInput<D, F>): Promise<EffectOutput<D, F>> }; // backend-only unless surface has exception
}

// React bindings
export function AppDomainProvider(props: { client: AppDomainClient<any>; surfaceId: string; children: React.ReactNode }): JSX.Element;
export function useAppDomain<D extends DomainDescriptor>(): AppDomainClient<D>;
// Convenience hooks built on @tanstack/react-query
export function useProcess<D, P extends ProcessKey<D>>(p: P): {
  run(input): Promise<ProcessResult<D,P>>; canRun: boolean; isRunning: boolean; error?: Error;
};
export function useEntityQuery<D, Q>(q: Q, args?): { data; isLoading; error };
```
- `useProcess("x").run(...)` is the canonical mutation path used in spec §10/§26.
- The SDK NEVER performs `fetch`/`WebSocket`/etc. directly. It delegates to a
  **transport** injected at construction: `createAppDomainClient({ transport, descriptor, surfaceId })`.
  `Transport` interface: `{ call(req: SdkRequest): Promise<SdkResponse> }`. The runtime
  (production) and preview runtime both implement `Transport`.
- `SdkRequest` matches spec §19.3 envelope exactly:
  ```ts
  interface SdkRequest {
    domainId: string; manifestVersion: string; surfaceId: string; surfaceDigest?: Sha256;
    actorId: string; tenantId: string;
    capability: "identity.current"|"permissions.can"|"entity.query"|"entity.get"|"process.run"|"event.subscribe"|"audit.annotate"|"effect.invoke";
    target?: string; input?: unknown; idempotencyKey?: string;
  }
  ```

### 3.2 Backend SDK
`createBackendDomainClient({ runtime, descriptor })` returning a server-side
`AppDomainClient` whose transport is the in-process runtime (no HTTP). Used by unit
process implementations.

## 4. App-Domain Runtime (`@echothink/app-domain-runtime`)

The runtime is the production authority and implements `Transport` (from §3.1). It is
constructed from a `CompiledManifest` + adapters and enforces every check in spec §19.

### 4.1 Components (each an interface + default impl)
```ts
DomainRegistryLoader   // loads compiled manifest + release metadata
IdentityContextResolver// resolves user/tenant/roles
PermissionEngine       // can(actor, permission, ctx) using permissionMatrix
EntityGateway          // query()/get() via EntityStore adapter, applies tenant+read policy
UnitProcessEngine      // run(process, input, ctx): permission -> validate input -> preconditions -> transition check -> writes (transactional) -> emit events -> audit -> effects
StateMachineEvaluator  // isLegalTransition(entity, field, from, to)
DomainEventBus         // emit/subscribe typed events
AuditEngine            // append(record): immutable log (AuditSink adapter)
EffectInvoker          // invoke(effectId, input, ctx): checks declared effect, egress, env, permission; calls EffectImpl
ReleaseGuard           // verify(req): release approved + surface digest match + sdk/runtime version match
```
Adapters (injected): `EntityStore`, `AuditSink`, `SecretResolver`, `EffectImpl[]`, `Clock`, `IdGenerator`.

### 4.2 Construction
```ts
export function createRuntime(opts: {
  compiled: CompiledManifest; release?: ReleaseManifest;
  entityStore: EntityStore; auditSink: AuditSink; secretResolver: SecretResolver;
  effects: EffectImpl<any,any>[]; clock: Clock; ids: IdGenerator;
}): DomainRuntime;
export interface DomainRuntime extends Transport { /* plus introspection getters */ }
```

### 4.3 Preview Runtime (spec §21)
```ts
export function createPreviewRuntime(opts: {
  compiled: CompiledManifest;
  fixtures: { personas: Persona[]; entities: Record<string, unknown[]>; effectStubs: EffectStub<any,any>[] };
  activePersonaId: string; clock: Clock; ids: IdGenerator;
}): PreviewRuntime;
export interface PreviewRuntime extends Transport {
  setPersona(id: string): void;
  inspectAudit(): AuditRecord[];
  inspectEvents(): EmittedEvent[];
  explainPermission(permission: string): { allowed: boolean; reason: string };
  forceFailure(kind: "permission"|"effect"|"runtime"|null): void;   // error boundary preview
}
```
Preview uses in-memory `EntityStore` seeded from fixtures, `EffectStub`s instead of
real effects, an in-memory audit/event log, and a permission engine identical to prod.
**Same checks, no real side effects.** This is what the workbench preview pane drives.

### 4.4 Audit record
```ts
interface AuditRecord { id; ts; actorId; tenantId; domainId; surfaceId?; capability;
  target?; result: "ok"|"denied"|"error"; reason?; redactedInput?: unknown; }
```

## 6. Validation pipeline (`@echothink/validation`)

Deterministic, repeatable locally and in CI. Produces `validation-report.json`.

```ts
export type GateId =
  | "manifest-schema" | "manifest-semantic" | "typescript" | "build"
  | "permission-simulation" | "entity-contract" | "process-contract"
  | "security-imports" | "dependency-allowlist" | "effect-simulation"
  | "storybook" | "accessibility" | "visual" | "release-completeness";
export interface GateResult {
  gate: GateId; status: "pass" | "fail" | "skip"; surfaceType?: SurfaceType;
  findings: Array<{ severity: "error"|"warning"; message: string; file?: string; line?: number; code: string }>;
  durationMs: number;
}
export interface ValidationReport {
  runId: string; domainId: string; version: string; createdAt: string;
  surfaceTypeProfile: Record<string, SurfaceType>;
  gates: GateResult[]; overall: "pass" | "fail";
}
export interface Gate { id: GateId; run(ctx: GateContext): Promise<GateResult>; }
export interface GateContext {
  domainDir: string; compiled: CompiledManifest; surfaces: SurfaceRegistration[];
  runId: string; now: string;
}
export function runPipeline(ctx: GateContext, gates?: GateId[]): Promise<ValidationReport>;
```
- **security-imports**: AST scan (use `@typescript-eslint/typescript-estree` or `ts`
  compiler API) of surface code; FAIL on the forbidden patterns in spec §18:
  `fetch(`, `axios`, `new WebSocket`, `new EventSource`, `localStorage.setItem("token"`,
  `process.env`, `@prisma/client`, `@supabase/supabase-js`, `node:fs`/`fs`, `eval(`,
  `new Function(`, and any import not in the surface's `allowedImports`.
- **permission-simulation**: for each surface & process, run allow + deny personas
  through a preview runtime; assert expected outcomes from the permission matrix.
- Gate applicability per surface type follows spec §25 matrix exactly (a table constant
  `GATE_MATRIX[surfaceType][gateId] = "required"|"conditional"|"skip"`).
- **CLI**: `validate:domain <domainDir> [--gates a,b] [--json out.json]` →
  exit code 0 on overall pass, 1 on fail. Entry `src/cli.ts`, bin name `echothink-validate`.

## 7. Agent harness (`@echothink/agent-harness`)

Controlled Codex runner. Enforces that Codex can propose/repair code but cannot bypass
governance (spec §15).

```ts
export interface FileScopePolicy { allowWriteGlobs: string[]; denyWriteGlobs: string[]; platformOwnedGlobs: string[]; }
export interface CommandPolicy { allow: string[]; }   // e.g. ["pnpm typecheck","pnpm test","pnpm build","echothink-validate"]
export interface HarnessPolicy { fileScope: FileScopePolicy; commands: CommandPolicy; network: "disabled"|"proxy"; allowNewDependencies: boolean; }
export interface RunArtifacts { runId; agent; domainId; inputArtifacts: string[]; outputArtifacts: string[]; claims: string[]; validationRequired: GateId[]; }
export interface CodexRunner { run(task: CodexTask): Promise<CodexRunResult>; }
export interface CodexTask { prompt: string; cwd: string; policy: HarnessPolicy; runId: string; }
export interface CodexRunResult { ok: boolean; patch: PatchSummary; logsPath: string; commandsRun: string[]; blockedActions: string[]; }
// Repair loop: generate -> typecheck -> build -> test -> validate -> (on fail) summarize errors -> repair prompt -> repeat up to maxIterations
export function runRepairLoop(opts: { runner: CodexRunner; validate(): Promise<ValidationReport>; task: CodexTask; maxIterations: number }): Promise<{ report: ValidationReport; iterations: number }>;
```
The default `CodexRunner` shells out to `codex exec --full-auto` with the file-scope and
command policy enforced by (a) a pre-run policy check on the prompt/cwd and (b) a
post-run diff check that REJECTS the patch if it touched deny/platform-owned globs or
added disallowed dependencies. All prompts, patches, command outputs saved under
`<domainDir>/.workbench/runs/<runId>/`.
